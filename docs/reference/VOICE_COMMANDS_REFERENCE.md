# 🎙️ Voice Commands Quick Reference

## Task Commands

### Creating Tasks
- "New task: [description]"
- "Add task: [description] for [project]"
- "Tomorrow: [task description]"
- "High priority: [task description]"

### Completing Tasks
- "Complete [task name]"
- "Done with [task name]"
- "Finished [task name]"

### Examples
✅ "New task: Call the dentist tomorrow"
✅ "Complete water the plants"
✅ "High priority: Submit expense report by Friday"
✅ "Add task: Review pull requests for backend project"

## Notes & Ideas

### Adding Notes
- "Note: [content]"
- "Idea: [content]"
- "Remember: [content]"
- "For [project]: [note content]"

### Examples
✅ "Note: Consider adding search feature to dashboard"
✅ "Idea for garden project: Install drip irrigation"
✅ "Remember: Team prefers morning standups"

## Time References

### Supported Formats
- "today", "tomorrow", "yesterday"
- "next Monday", "this Friday"
- "in 3 days", "next week"
- "by end of week", "by month end"

### Examples
✅ "New task: Prepare slides by next Tuesday"
✅ "Complete report today"
✅ "Schedule meeting for next week"

## Project References

### Linking to Projects
- "For [project name]: [task/note]"
- "[task] in [project]"
- "[project] task: [description]"

### Examples
✅ "For website redesign: Create wireframes"
✅ "Backend project task: Fix authentication bug"
✅ "Garden project note: Roses need pruning in spring"

## Tips for Clear Recording

### DO ✅
- Speak clearly and at normal pace
- Pause briefly between different items
- State the action first (new task, complete, note)
- Keep each recording under 2 minutes
- Hold watch closer to mouth in noisy environments

### DON'T ❌
- Rush through multiple items
- Include background conversations
- Use ambiguous pronouns without context
- Record while walking in windy conditions

## Multi-Item Recording

### Format
"First, [command 1]. Next, [command 2]. Also, [command 3]."

### Example
"First, complete the invoice task. Next, new task: Follow up with client next week. Also, note: Client prefers email communication."

## Ambiguous Content
These will go to review queue:
- General observations without actions
- Unclear references
- Personal thoughts
- Partial sentences

### Examples
🔍 "Feeling good about project progress"
🔍 "That thing we discussed yesterday"
🔍 "Maybe we should consider..."
🔍 "The weather is nice today"
